The Problem

Developers use Make for tasks, GitHub Actions for CI, shell scripts for releases.
AI agents read JSON, humans want pretty output.
You need two tools. Fledge is one.

Six Pillars

Scaffold

Start any project from a template. Language-aware, git-ready, zero config.

Run

One command to build, test, lint — auto-detects your stack.

Spec

Keep specs and code in sync. Drift detected, never ignored.

AI

Code review, Q&A, and generation — built in, not bolted on.

Ship

Bump, changelog, tag, release. One command from commit to publish.

Extend

Plugins and lanes — compose your own workflows.

Same Tool. Different Roles.

Developer

$ fledge run test
$ fledge review
$ fledge release patch

Interactive prompts, pretty diffs, human-friendly output.

AI Agent

$ fledge run test --output json
$ fledge review --output json
$ fledge release patch --yes

Structured output, no prompts, machine-parseable.

New Hire

$ fledge init
$ fledge doctor
$ fledge run dev

Guided setup, environment checks, zero tribal knowledge required.

Same tool. Different flags. Everyone wins.

Build On It

Extend with what you need. Fledge plugins are just Rust crates — publish, share, compose.

# Install a community plugin
$ fledge plugin add coverage

# Use it immediately
$ fledge coverage report
